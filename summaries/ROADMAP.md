# ROADMAP — iSystem Development (2026)

High-level phases. Adjust dates in [MILESTONES.md](MILESTONES.md) as work lands.

---

## Phase 1 — Platform SSOT & web polish *(current)*

- Org docs, profile, wiki, and GitHub Pages aligned with DIODAC branding.
- iMETER v2 hero, data console, cache versioning deployed to VPS.
- Cloudflare cache rules documented and applied per [cache standard](https://github.com/iSystemDevelopment/isystem-deploy/blob/main/docs/cloudflare-cache-standard.md).

## Phase 2 — Portfolio & product surfaces

- **diodac.org** live on VPS with DNS.
- Pin flagship repos on org GitHub profile.
- WordPress repo structure and plugin docs stable for Spectra companion theme.

## Phase 3 — Instrumentation & integrations

- Live device webhook path for iMETER (bench → JSON → data console).
- WireViz / certificate pipeline improvements.
- Business Suite inventory & BOM tooling (new implementation on iSystem platform).

## Phase 4 — Cloud & API *(deferred)*

- **isystem.cloud** services and public API surface.
- Multi-tenant API hardening and observability.
