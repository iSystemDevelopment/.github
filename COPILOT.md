# GitHub Copilot Usage Guidelines

This repository uses GitHub Copilot and other AI assistants to maintain code quality and consistency.

## For AI Assistants

When working in this repository:

1. **Privacy First**: Do not expose users' original prompts or unpolished requests in public PR descriptions or commits
2. **Professional Tone**: All public-facing documentation should use professional, technical language
3. **Clear Communication**: PR descriptions should clearly explain what was changed and why
4. **Quality Standards**: Follow established code quality, security, and testing practices
5. **Consistency**: Maintain consistency with existing patterns and conventions

## For Contributors

This repository has been configured to work seamlessly with AI-powered development tools:

- GitHub Copilot provides code suggestions and assistance
- AI assistants help maintain documentation and code quality
- Automated checks ensure consistency and security

All AI-generated contributions follow the same quality standards as human contributions.

## Configuration

The repository includes private configuration files that guide AI assistant behavior. These files are local-only and not committed to version control to protect privacy and internal workflows.

For more information on using Copilot in your projects, see the [official documentation](https://docs.github.com/en/copilot).
